## UP-2: BoundarySafeSearchEngine (L3) — 분리 모듈

**엔진 번호**: UP-2  
**역할**: UP-1의 L2 출력(`SearchBias`) 위에서 **안전 영역 탐색**(L3)을 수행하는 별도 엔진  
**상태**: 설계 완료, 구현 대기

---

### 왜 분리인가?

- **UP-1(원인 분석) 정체성 보호**: UP-1은 "왜 실패하는지"만 다루고, 해결/탐색은 섞지 않음
- **오해 방지**: L3가 들어가면 "삼체 문제를 푼다"로 오해될 리스크 증가
- **아키텍처 규칙**: *진단서(원인)와 처방전(탐색/해결)은 분리*

---

### 입력/출력 (UP-1과의 경계)

- **입력**: `SearchBias` (UP-1 L2 결과)
- **출력**: `SafePath`(가칭) — 위험 지형 위에서 붕괴하지 않는 경로 후보

---

### ⚠️ 다른 난제와의 관계

**UP-2는 나비효과/스토크스와 무관합니다.**

- **나비효과 (Butterfly Effect)**: 초기 조건 민감성 분석 → **별도 엔진 (UP-3 등)**
- **스토크스 (Navier-Stokes)**: 유체 경계 형성 분석 → **별도 엔진 (UP-4 등)**

**UP-2의 역할**:
- UP-1이 만든 **위험 지형** 위에서만 탐색
- 삼체 문제든 다른 문제든, **SearchBias만 입력**으로 받으면 동작 가능
- 하지만 나비효과/스토크스 자체를 분석하는 건 아님
