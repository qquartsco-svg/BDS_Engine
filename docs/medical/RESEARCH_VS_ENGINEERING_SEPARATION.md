# 의료 연구용 vs 엔지니어링 관점 분리 전략

**작성일**: 2025-01-26  
**목적**: 의료 연구용 시뮬레이션과 엔지니어링 관점 코드의 명확한 분리

---

## 🎯 핵심 원칙

### 의료 연구용 시뮬레이션
- **목적**: 실제 우울증 연구 자료로 의미있는 분석 결과 도출
- **기준**: 생물학적 타당성, 임상적 관련성, 연구 재현성
- **출력**: 연구 논문에 사용 가능한 데이터, 통계적 유의성

### 엔지니어링 관점 코드
- **목적**: 동역학 메커니즘 이해, 시스템 설계, 확장성
- **기준**: 코드 품질, 성능, 아키텍처, 재사용성
- **출력**: 확장 가능한 엔진, 모듈화된 구조

---

## 📊 분리 전략

### 1. 디렉토리 구조 분리

```
brain_disorder_simulation/
├── research/                    # 의료 연구용 (NEW)
│   ├── depression/
│   │   ├── clinical_models.py   # 임상 모델
│   │   ├── biomarkers.py        # 생체지표 매핑
│   │   ├── validation.py        # 검증 도구
│   │   └── analysis.py          # 연구 분석 도구
│   └── utils/
│       ├── statistical.py       # 통계 분석
│       └── reporting.py          # 연구 리포트
│
├── engineering/                 # 엔지니어링 관점 (NEW)
│   ├── dynamics/                # 동역학 엔진
│   ├── architecture/            # 시스템 아키텍처
│   └── optimization/            # 성능 최적화
│
└── core/                        # 공통 코어 (기존)
    ├── common/                  # 공통 엔진
    ├── disorders/              # 질환별 엔진
    └── unified/                 # 통합 시뮬레이터
```

### 2. 모델 분리

#### 의료 연구용 모델
- **입력**: 임상 데이터, 생체지표 (fMRI, EEG, HRV)
- **파라미터**: DSM-5/ICD-11 기준, 임상 스케일
- **출력**: 연구 논문용 데이터, 통계 분석, 임상 해석

#### 엔지니어링 관점 모델
- **입력**: 시스템 상태, 동역학 파라미터
- **파라미터**: 엔지니어링 스케일, 시스템 변수
- **출력**: 동역학 분석, 시스템 설계 인사이트

## 🔬 의료 연구용 요구사항

### 1. 생물학적 타당성

**필수 요소:**
- 뇌 영역 매핑 정확성
- 신경전달물질 시스템 (도파민, 세로토닌, 노르에피네프린)
- 시간 스케일 (실제 뇌 활동과 일치)
- 에너지 대사 모델

**검증 방법:**
- 기존 연구 자료와 비교
- 생물학적 제약 조건 검증
- 물리적 타당성 테스트

### 2. 임상적 관련성

**필수 요소:**
- DSM-5/ICD-11 기준 매핑
- 임상 스케일 (HAM-D, BDI, PHQ-9)
- 증상 패턴 재현
- 개인차 모델링

**검증 방법:**
- 실제 환자 데이터와 비교
- 임상 전문가 검토
- 통계적 유의성 검증

### 3. 연구 재현성

**필수 요소:**
- Seed 관리
- 실험 메타데이터
- 파라미터 문서화
- 결과 추적성

**검증 방법:**
- 재현성 테스트
- 메타데이터 검증
- 버전 관리
